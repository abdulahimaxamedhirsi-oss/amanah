1. OBJECTIF DU SYSTÈME

Le Multi-Tenant Engine permet à Amanah d'héberger plusieurs organisations indépendantes sur une infrastructure commune tout en garantissant une isolation complète des données, des configurations et des ressources.

Un tenant peut être :

une banque
une fintech
un PSP
une institution de microfinance
un opérateur Mobile Money
une entreprise
une administration
un environnement Sandbox
un partenaire OEM

Chaque tenant fonctionne comme un environnement totalement indépendant.

2. CORE PRINCIPLES
2.1 Complete Tenant Isolation
No tenant can access another tenant's resources.

L'isolation concerne :

données
utilisateurs
APIs
stockage
événements
clés
configurations
journaux
2.2 Shared Infrastructure
Infrastructure is shared.
Business data is never shared.

Infrastructure commune :

Kubernetes
Monitoring
CI/CD
Observability
AI Gateway
Notification Engine
Backup
Disaster Recovery
2.3 Tenant Context Everywhere
Every request must include a tenant context.

Aucune donnée ne peut traverser la frontière d'un tenant.

19. CLAUDE CODE RULES

Claude Code must:

ALWAYS require a tenantId before processing requests.
NEVER expose resources across tenants.
ALWAYS isolate secrets, storage and configurations.
NEVER allow shared authentication contexts.
ALWAYS propagate tenant context to downstream services.
NEVER execute background jobs without tenant ownership.
ALWAYS support multiple database isolation strategies.
NEVER mix observability data between tenants.
ALWAYS enforce tenant-aware RBAC and ABAC.
ALWAYS support automated tenant provisioning and migration.
20. SUCCESS CRITERIA

Le système Multi-Tenant est considéré comme complet lorsque :

chaque organisation fonctionne dans un environnement isolé ;
les données, configurations et secrets sont strictement séparés ;
plusieurs stratégies d'isolation des bases de données sont supportées ;
le provisionnement et la migration des tenants sont automatisés ;
toutes les requêtes sont contextualisées par tenantId et entityId ;
la plateforme peut héberger des centaines ou des milliers de tenants sans compromettre la sécurité ou les performances.
