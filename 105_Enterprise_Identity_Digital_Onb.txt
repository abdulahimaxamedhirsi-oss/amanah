1. OBJECTIF

Ce document définit le framework officiel de gestion de l'identité numérique et de l'onboarding d'Amanah.

Il couvre :

Digital Identity ;
Digital Onboarding ;
eKYC ;
KYB ;
Identity Verification ;
Document Verification ;
Biometric Verification ;
Risk-Based Authentication ;
Customer Identity Lifecycle ;
Identity Governance.

L'objectif est d'offrir un processus d'identification rapide, sécurisé et conforme aux réglementations internationales.

2. CORE PRINCIPLES
2.1 Digital First

Identity verification should be fully digital whenever regulations allow.

2.2 Trust by Verification

Customer trust is established through verifiable identity evidence.

2.3 Continuous Identity

Identity verification continues throughout the customer relationship.

7. DOCUMENT VERIFICATION

Documents acceptés :

Passeport ;
Carte nationale d'identité ;
Permis de conduire ;

8. OCR ENGINE

Extraction automatique :

nom ;
prénom ;
date de naissance ;
numéro de document ;
date d'expiration ;
nationalité ;
adresse (si présente).

Les données extraites sont validées avant utilisation.

9. BIOMETRIC VERIFICATION

Support :

reconnaissance faciale ;
empreinte digitale (si disponible) ;
reconnaissance vocale (optionnelle) ;
selfie sécurisé.
10. LIVENESS DETECTION

Contrôles :

présence réelle ;
anti-photo ;
anti-vidéo ;
anti-deepfake ;
détection de présentation frauduleuse.
11. IDENTITY MATCHING

Le moteur compare :

selfie ↔ document ;
biométrie ↔ historique ;
identité ↔ bases officielles (si disponibles).

Chaque comparaison reçoit un score de confiance.

12. eKYC ENGINE

Le moteur eKYC vérifie :

identité ;
âge ;
résidence ;
activité professionnelle ;
origine des fonds (si applicable).
13. KYB (KNOW YOUR BUSINESS)

26. CLAUDE CODE RULES

Claude Code must:

ALWAYS verify customer identity before account activation.
NEVER approve onboarding when mandatory KYC requirements are incomplete.
ALWAYS validate identity documents using configurable verification rules.
ALWAYS perform biometric verification where required.
ALWAYS integrate AML and sanctions screening before account approval.
ALWAYS maintain immutable onboarding audit trails.
ALWAYS support continuous KYC throughout the customer lifecycle.
ALWAYS protect identity documents using encryption and strict access controls.
ALWAYS expose onboarding services through secure APIs.
ALWAYS follow Amanah Enterprise Identity Standards.
27. SUCCESS CRITERIA

Le framework Identity & Digital Onboarding est conforme lorsque :

un client peut ouvrir un compte entièrement à distance ;
toutes les vérifications KYC et AML sont effectuées avant activation ;
les documents et données biométriques sont protégés ;
les revues KYC sont automatisées tout au long du cycle de vie du client ;
chaque décision d'onboarding est entièrement traçable et auditée.
