1. OBJECTIF DU SYSTÈME

Ce document définit toute la stratégie de sécurité d’Amanah.

👉 Objectif :

empêcher toute fraude

2. CORE SECURITY PRINCIPLE
2.1 Zero Trust Architecture
Never trust user, device, network, or service by default
2.2 Defense in depth

Sécurité en couches :

API layer security
service layer validation
ledger enforcement
database protection
infrastructure security
2.3 Ledger is untouchable
Ledger = IMMUTABLE CORE
Any compromise attempt = system failure prevention trigger
3. AUTHENTICATION SECURITY
3.1 JWT protection

device-bound sessions (optional)
3.2 Session hijacking prevention
IP fingerprinting

anomaly detection
4. API SECURITY
4.1 Rate limiting
per user
per IP
per endpoint
4.2 Abuse protection

Block:

brute force
endpoint scraping
bot traffic
payment spam
4.3 Input validation
strict DTO validation
schema enforcement
no raw input passing
5. ENCRYPTION SYSTEM
5.1 Data at rest
AES-256 encryption mandatory
sensitive fields encrypted in DB
5.2 Data in transit
TLS 1.3 required
certificate pinning for mobile apps
5.3 Key management
environment secrets never exposed
rotated regularly
stored in secure vault (e.g. AWS KMS)

6.1 Real-time detection

6.2 Fraud response levels
monitor

freeze account
