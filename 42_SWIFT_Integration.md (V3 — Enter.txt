1. OBJECTIF DU SYSTÈME

Ce document définit l'architecture complète permettant à Amanah de communiquer avec le réseau bancaire mondial via SWIFT.

L'objectif est de rendre Amanah capable de :

envoyer des paiements internationaux
recevoir des paiements internationaux
communiquer avec les banques partenaires
suivre les paiements en temps réel
respecter les standards SWIFT et ISO 20022
intégrer SWIFT gpi
assurer une traçabilité bancaire complète

Le système ne remplace pas le Ledger.

Le Ledger reste la source de vérité.

SWIFT est uniquement un réseau de messagerie financière.

2. CORE PRINCIPLE
2.1 SWIFT is messaging, not settlement
SWIFT transports financial instructions.
It does not move money itself.

L'argent est déplacé :

via les comptes de règlement
les banques correspondantes
les banques centrales
les systèmes de compensation

SWIFT ne transporte que les messages.

2.2 Ledger Independence
The Ledger must never depend on SWIFT availability.

Si SWIFT est indisponible :

le Ledger continue de fonctionner
les Wallets continuent
les paiements internes continuent

Seuls les paiements SWIFT sont suspendus.

19. CLAUDE CODE RULES

Claude Code must:

NEVER connect directly to the SWIFT Network outside the SWIFT Gateway.
ALWAYS validate BIC, IBAN and currency before message creation.
NEVER mark a payment as settled without external confirmation.
ALWAYS generate and preserve the UETR for every international payment.
NEVER allow duplicate SWIFT message transmission.
ALWAYS archive the original and signed message.
NEVER expose private keys or certificates.
ALWAYS maintain full traceability from Payment ID to SWIFT UETR.
NEVER couple Ledger logic to SWIFT protocols.
ALWAYS support migration from MT messages to ISO 20022 MX.
20. SUCCESS CRITERIA

Le module SWIFT est considéré comme terminé lorsque :

les paiements internationaux sont entièrement compatibles avec le réseau SWIFT ;
les messages MT et ISO 20022 peuvent être pris en charge selon les partenaires ;
chaque paiement dispose d'un UETR et d'une traçabilité complète ;
les validations BIC, IBAN, conformité et signatures sont systématiques ;
le Ledger reste totalement indépendant du réseau SWIFT ;
les communications sont sécurisées, auditables et hautement disponibles.
