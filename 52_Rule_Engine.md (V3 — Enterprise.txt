52_Rule_Engine.md (V3 — Enterprise Business Rule Engine)
1. OBJECTIF DU SYSTÈME

Le Rule Engine est le moteur central des règles métier d'Amanah.

Il permet de modifier le comportement de la plateforme sans redéploiement.

Toutes les décisions métier doivent être pilotées par des règles configurables.

Le Rule Engine ne contient aucune donnée métier.

Il prend des décisions.

2. CORE PRINCIPLES
2.1 Configuration over Code
Business behavior must be configurable without changing source code.
2.2 Rules are Data
Every business rule is stored as versioned configuration.
2.3 Deterministic Decisions
The same input must always produce the same decision.

20. CLAUDE CODE RULES

Claude Code must:

NEVER hardcode business rules inside application code.
ALWAYS evaluate rules through the Rule Engine.
NEVER bypass critical regulatory rules.
ALWAYS support rule versioning.
NEVER modify active rules without audit logs.
ALWAYS support simulation mode.
ALWAYS evaluate rules deterministically.
NEVER execute conflicting rules without priority resolution.
ALWAYS support country-specific rule sets.
ALWAYS maintain complete traceability for every decision.
21. SUCCESS CRITERIA

Le Rule Engine est considéré comme complet lorsque :

toutes les décisions métier passent par des règles configurables ;
les règles sont versionnées, auditables et rechargeables à chaud ;
les politiques varient par pays, entité ou tenant sans modification du code ;
les règles peuvent être testées en mode simulation avant publication ;
chaque décision est traçable et reproductible.
22. FUTURE EXTENSIONS (V4 READY)

Le Rule Engine est conçu pour intégrer ultérieurement :

Policy-as-Code avec validation automatique dans les pipelines CI/CD.
Natural Language Rule Builder, où un administrateur peut écrire une règle en langage naturel et le moteur la convertit en règle exécutable.
