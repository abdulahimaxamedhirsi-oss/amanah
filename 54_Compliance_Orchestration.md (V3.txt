1. OBJECTIF DU SYSTÈME

La Compliance Orchestration Platform est le centre de contrôle réglementaire d'Amanah.

Elle orchestre tous les moteurs de conformité afin de garantir que chaque client, paiement et opération respecte les exigences réglementaires internationales.

Le moteur ne remplace pas les systèmes KYC ou AML.

Il les coordonne.

2. CORE PRINCIPLES
2.1 Compliance by Design
Every financial operation must be compliant before execution.
2.2 Centralized Compliance

Tous les contrôles passent par une seule plateforme.

Aucun microservice ne réalise ses propres vérifications AML.

2.3 Event Driven Compliance

Chaque événement déclenche automatiquement les contrôles nécessaires.

22. CLAUDE CODE RULES

Claude Code must:

ALWAYS orchestrate compliance checks through the Compliance Orchestration Platform.
NEVER bypass AML, KYC or sanctions screening.
ALWAYS execute mandatory regulatory checks before settlement.
NEVER allow AI to make final compliance decisions without human approval where required.
ALWAYS support jurisdiction-specific compliance workflows.
ALWAYS maintain immutable audit logs.
ALWAYS support continuous monitoring after onboarding.
NEVER allow outdated sanctions lists.
ALWAYS generate regulatory reports when required.
ALWAYS ensure every compliance decision is traceable and reproducible.
23. SUCCESS CRITERIA

La Compliance Orchestration Platform est considérée comme complète lorsque :

tous les moteurs de conformité sont centralisés et orchestrés ;
les contrôles sont exécutés automatiquement selon les événements métier ;
les réglementations internationales (KYC, AML, PEP, Sanctions, FATCA, CRS, Travel Rule) sont prises en charge ;
les dossiers d'enquête sont gérés de bout en bout ;
toutes les décisions sont auditables, versionnées et reproductibles ;
la plateforme peut être adaptée à chaque juridiction sans modifier le code.
