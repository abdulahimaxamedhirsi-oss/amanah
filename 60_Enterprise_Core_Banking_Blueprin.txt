1. PURPOSE

Le Enterprise Core Banking Blueprint définit l'architecture globale d'Amanah.

Il constitue la référence unique décrivant :

la vision du Core Banking ;
les principes d'architecture ;
les composants ;
les interactions ;
les standards ;
les exigences de sécurité ;
les exigences réglementaires ;
les exigences de performance ;
la feuille de route.

Toute évolution du système doit respecter ce document.

2. VISION

Amanah est une plateforme Core Banking Cloud Native, AI-Native, API-First, Event-Driven, Multi-Tenant, conçue pour servir :

banques commerciales
banques islamiques
institutions de microfinance
opérateurs Mobile Money
fintechs
gouvernements
grandes entreprises

sur une infrastructure mondiale hautement disponible.

19. CLAUDE CODE RULES

Claude Code must:

ALWAYS comply with this Enterprise Core Banking Blueprint.
NEVER introduce architectural inconsistencies.
ALWAYS preserve domain boundaries.
ALWAYS use shared enterprise platforms instead of duplicating functionality.
ALWAYS enforce security, compliance and audit requirements.
NEVER bypass the Ledger, Workflow, Rule Engine or Policy Engine.
ALWAYS design new features as modular, reusable services.
ALWAYS maintain backward compatibility unless a documented migration exists.
ALWAYS document architectural decisions (ADR) for significant changes.
ALWAYS prioritize maintainability, resilience and regulatory compliance over short-term implementation speed.
20. SUCCESS CRITERIA

L'Enterprise Core Banking Blueprint est considéré comme complet lorsque :

tous les composants de la plateforme sont définis et leurs interactions documentées ;
les principes architecturaux sont appliqués de manière cohérente dans tous les domaines ;
la sécurité, la conformité, la résilience et l'observabilité sont intégrées par conception ;
la plateforme est capable d'évoluer sans remise en cause de son architecture ;
ce document sert de référence unique pour les architectes, développeurs, intégrateurs et auditeurs.
