1. OBJECTIF DU SYSTÈME

Le Treasury Management System (TMS) est responsable de la gestion centralisée de la trésorerie d'Amanah.

Il supervise :

la liquidité globale
les positions par devise
les positions par pays
les comptes bancaires
les comptes de règlement
les comptes Nostro/Vostro
les besoins de financement
les réserves réglementaires
les prévisions de trésorerie

Le TMS ne traite jamais directement les paiements.

Il fournit les ressources nécessaires aux autres moteurs financiers.

2. CORE PRINCIPLES
2.1 Liquidity is a Strategic Asset
Every unit of liquidity must be visible, measurable and optimally allocated.
2.2 Single Source of Treasury Truth
All treasury positions must originate from reconciled Ledger balances.

Le TMS ne maintient jamais un solde indépendant du Ledger.

2.3 Capital Preservation
Treasury operations must prioritize capital preservation over yield.

18. SECURITY

Toutes les opérations de trésorerie doivent être :

authentifiées
autorisées selon le principe des quatre yeux (Four-Eyes Principle) pour les opérations critiques
signées si nécessaire
journalisées
auditables
protégées contre les modifications non autorisées

Aucune opération critique de trésorerie ne peut être exécutée par une seule personne.

19. CLAUDE CODE RULES

Claude Code must:

NEVER calculate treasury balances independently from the Ledger.
ALWAYS validate liquidity before funding decisions.
NEVER execute treasury reallocations without policy validation.
ALWAYS maintain positions by currency, country, entity and institution.
NEVER allow treasury operations without audit logs.
ALWAYS preserve regulatory reserves.
NEVER exceed configured liquidity limits.
ALWAYS synchronize treasury positions with the Settlement Network.
NEVER expose sensitive banking account information.
ALWAYS support forecasting, stress testing and scenario analysis.
20. SUCCESS CRITERIA

Le Treasury Management System est considéré comme complet lorsque :

la liquidité est visible en temps réel à tous les niveaux (global, pays, devise, banque, entité) ;
les positions sont dérivées du Ledger et réconciliées automatiquement ;
les réserves réglementaires sont protégées ;
les besoins futurs de liquidité sont anticipés ;
les décisions de financement et de réallocation sont gouvernées par des politiques configurables ;
toutes les opérations sont traçables, sécurisées et conformes aux exigences d'audit.
