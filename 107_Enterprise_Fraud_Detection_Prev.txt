(V3 — Enterprise Fraud Detection, Prevention, Behavioral Analytics & Financial Crime Protection Framework)
1. OBJECTIF

Ce document définit le framework officiel de lutte contre la fraude d'Amanah.

Il couvre :

Fraud Detection ;
Fraud Prevention ;
Real-Time Fraud Scoring ;
Behavioral Biometrics ;
Device Intelligence ;
Account Takeover (ATO) Detection ;
Payment Fraud ;
Card Fraud ;
Mule Account Detection ;
Fraud Case Management.

L'objectif est de détecter, prévenir et limiter les fraudes tout en réduisant les faux positifs.

2. CORE PRINCIPLES
2.1 Prevention First

Prevent fraud before financial loss occurs.

2.2 Risk Adaptive

Fraud controls adapt dynamically to contextual risk.

2.3 Customer Experience

Security must minimize friction for legitimate customers.

7. BEHAVIORAL BIOMETRICS

Analyse :

vitesse de frappe ;
mouvements de souris ;
gestes tactiles ;

habitudes d'utilisation.

Ces signaux enrichissent le score de fraude.

8. ACCOUNT TAKEOVER (ATO)

Détection :

changement brutal d'appareil ;
nouvelle localisation ;
changement d'identifiants ;
tentatives de connexion répétées ;
comportement inhabituel.

Les comptes suspects peuvent être temporairement protégés.

10. CARD FRAUD

Card Not Present (CNP) ;

26. CLAUDE CODE RULES

Claude Code must:

ALWAYS evaluate fraud risk before authorizing high-risk operations.
ALWAYS combine rule-based and AI-assisted fraud detection.
NEVER allow fraud decisions to bypass audit logging.
ALWAYS support adaptive authentication based on risk.
ALWAYS correlate fraud signals across channels and products.
ALWAYS maintain immutable fraud investigation histories.
ALWAYS minimize customer friction for low-risk activities.
ALWAYS integrate fraud controls with payments, cards and identity services.
ALWAYS monitor fraud model performance and false-positive rates.
ALWAYS follow Amanah Enterprise Fraud Prevention Standards.
27. SUCCESS CRITERIA

Le framework Fraude est conforme lorsque :

les fraudes sont détectées avant la perte financière lorsque cela est possible ;
les faux positifs restent maîtrisés ;
les décisions sont explicables et auditables ;
les réponses sont adaptées au niveau de risque ;
tous les canaux (mobile, web, cartes, paiements, API) sont protégés de manière cohérente.
