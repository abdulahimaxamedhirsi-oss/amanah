L'objectif est de produire automatiquement des rapports réglementaires fiables, auditables et conformes aux exigences des autorités de supervision.

2. CORE PRINCIPLES
2.1 Single Source of Truth
All regulatory reports must originate from governed and reconciled data sources.
2.2 Automation by Default
Regulatory reporting should be automated wherever possible.
2.3 Full Traceability
Every reported figure must be traceable back to its originating transaction.

25. CLAUDE CODE RULES

Claude Code must:

ALWAYS generate regulatory reports from reconciled data.
NEVER allow manual modification of validated regulatory figures without audit.
ALWAYS validate reports before submission.
ALWAYS preserve report versions and submission history.
ALWAYS support XBRL where required.
ALWAYS maintain traceability from reported values to source transactions.
ALWAYS integrate reporting with risk, finance and ledger modules.
ALWAYS enforce approval workflows before submission.
ALWAYS archive submitted reports immutably.
ALWAYS follow Enterprise Regulatory Reporting Standards.
26. SUCCESS CRITERIA

Le framework de reporting réglementaire est conforme lorsque :

les rapports sont générés automatiquement ;
toutes les données sont traçables jusqu'à leur origine ;
les validations détectent les incohérences avant soumission ;
les transmissions sont sécurisées et auditables ;
les exigences des régulateurs sont satisfaites sans intervention manuelle excessive.
