1. OBJECTIF DU DOCUMENT

Ce document définit toute l’expérience utilisateur d’Amanah :

Mobile App (principal)
Web App (dashboard)
Agent App (M-Amanah)
Merchant Dashboard

👉 Ce n’est pas du “design”.
👉 C’est une spécification d’interface financière exécutable par développeurs.

2. UX PHILOSOPHY (FINTECH-FIRST DESIGN)

Amanah UX repose sur 5 principes :

2.1 Clarity over beauty

Chaque écran doit répondre à :

“Où est mon argent ?”

2.2 Financial transparency

L’utilisateur doit toujours voir :

balance totale
balance disponible
transactions récentes
2.3 Zero ambiguity

Aucune action ne doit être interprétée.

Exemple :

“Send” = immédiatement clair
pas de double sens
pas de jargon bancaire complexe
2.4 Instant feedback

Toute action doit produire :

loading state
success state
failure state
2.5 Africa-first UX

Optimisé pour :

low bandwidth
mobile-first
offline resilience
simple navigation

9. FINANCIAL UX RULES

9.2 No ambiguous balance

Pending balance

13. ANIMATION RULES
only meaningful animations

15. CLAUDE CODE RULES

Claude Code must:

never break financial clarity in UI
always display fees explicitly
never hide FX conversions
ensure wallet always matches ledger
implement all states (loading/success/error)
16. SUCCESS CRITERIA

zero ambiguous UI flows
full real-time sync with backend
no hidden financial operations
