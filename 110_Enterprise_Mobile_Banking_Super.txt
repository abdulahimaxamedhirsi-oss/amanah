(V3 — Enterprise Mobile Banking, Super App, Digital Experience & Omnichannel Framework)
1. OBJECTIF

Ce document définit le framework officiel de la Super App Amanah.

Il couvre :

Mobile Banking ;
Super App ;
Omnichannel Experience ;
Design System ;
Digital Wallet ;
Personal Finance Management (PFM) ;
Embedded Finance ;
AI Personalization ;
Offline Capabilities ;
Accessibility.

L'objectif est d'offrir une expérience bancaire mobile moderne, intuitive, sécurisée et hautement performante.

2. CORE PRINCIPLES
2.1 Mobile First

Every feature must be designed for mobile before desktop.

2.2 User Experience First

Customer experience is a primary business capability.

2.3 Real-Time Experience

Users should receive immediate feedback for every financial operation.

26. CLAUDE CODE RULES

Claude Code must:

ALWAYS prioritize mobile-first user experience.
ALWAYS keep the interface responsive across supported devices.
ALWAYS use the shared Design System.
NEVER expose sensitive information without strong authentication.
ALWAYS synchronize financial data securely and efficiently.
ALWAYS support biometric authentication and Passkeys where available.
ALWAYS optimize performance for common banking operations.
ALWAYS provide accessible interfaces compliant with WCAG standards.
ALWAYS integrate all customer actions through secure APIs.
ALWAYS follow Amanah Enterprise Mobile Banking Standards.
27. SUCCESS CRITERIA

Le framework Mobile Banking est conforme lorsque :

l'application offre une expérience fluide sur tous les appareils pris en charge ;
les opérations financières sont rapides et sécurisées ;
les fonctionnalités sont cohérentes entre les différents canaux ;
les normes d'accessibilité sont respectées ;
l'application reste évolutive grâce à son architecture modulaire.
