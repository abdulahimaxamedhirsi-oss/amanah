1. OBJECTIF DU SYSTÈME

Ce document définit le tableau de bord administrateur d’Amanah.

👉 Objectif :

supervision totale du système
contrôle des utilisateurs
surveillance des transactions
gestion conformité (KYC/AML)
audit du Ledger
gestion des incidents
2. CORE PRINCIPLE
2.1 Admin is observer + controller (not god)
Admin can manage operations but NEVER modify financial truth (Ledger)
2.2 Everything is traceable
every admin action is logged
every decision has traceId
every change is auditable
3. ADMIN DASHBOARD MODULES
3.1 Overview Dashboard
total users
total transactions
daily volume
failed payments
risk alerts
3.2 Real-time system health
API status
payment engine status
FX engine status
ledger sync status
4. USER MANAGEMENT PANEL
4.1 Features
search users
view profile
freeze/unfreeze account
view wallet summary (read-only ledger projection)
4.2 Rules
Admin cannot directly edit wallet or ledger
5. TRANSACTION MONITORING PANEL
5.1 Features
live transaction feed
filter by status (success, failed, pending)
view transaction trace
5.2 Transaction drill-down

Each transaction shows:

user sender/receiver
FX conversion details
fees breakdown
ledger batch reference
6. LEDGER AUDIT PANEL (CRITICAL)
6.1 Features
view ledger batches
verify double-entry consistency
detect anomalies
compare expected vs actual balances
6.2 Rule
Ledger is read-only in admin panel

7.1 KYC review
