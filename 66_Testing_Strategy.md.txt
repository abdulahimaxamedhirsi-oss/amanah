66_Testing_Strategy.md
(V3 — Enterprise Quality Assurance & Testing Framework)
1. OBJECTIF

Ce document définit la stratégie de tests obligatoire pour tous les composants Amanah.

Il garantit :

qualité logicielle ;
stabilité ;
sécurité ;
conformité ;
performance ;
fiabilité financière.

Aucun composant critique ne peut être déployé sans validation complète.

2. CORE PRINCIPLES
2.1 Quality by Design
Testing begins during architecture and development, not after coding.

La qualité est intégrée dans tout le cycle de vie.

2.2 Automation First
Critical tests must run automatically in CI/CD.

Les tests manuels complètent l'automatisation mais ne la remplacent pas.

2.3 Risk-Based Testing

Les tests sont prioritaires selon le risque métier.

Exemple :

Ledger :

★★★★★ Critical

Notifications :

★★ Medium

27. CLAUDE CODE RULES

Claude Code must:

ALWAYS generate tests with new features.
NEVER deliver untested financial logic.
ALWAYS test edge cases.
ALWAYS protect Ledger correctness.
ALWAYS include integration tests for critical workflows.
ALWAYS validate API contracts.
ALWAYS consider security testing.
NEVER ignore failed tests.
ALWAYS maintain regression coverage.
ALWAYS follow Enterprise Testing Standards.
28. SUCCESS CRITERIA

La stratégie de tests est respectée lorsque :

chaque fonctionnalité possède une couverture adaptée ;
les flux financiers critiques sont validés ;
les régressions sont détectées automatiquement ;
la sécurité et la performance sont testées avant production ;
les releases sont prévisibles et contrôlées.
