1. OBJECTIF DU SYSTÈME

Le Payment Engine est responsable de :

Orchestrer tous les mouvements d’argent dans Amanah de manière atomique, traçable et sécurisée.

Rôle principal
exécuter les transferts entre utilisateurs
déclencher FX si nécessaire
appliquer les fees
écrire dans le Ledger
garantir atomicité complète
gérer les erreurs et rollback
2. CORE PRINCIPLE
2.1 Payment = orchestrated workflow
Payment ≠ simple transfer
Payment = multi-step financial orchestration pipeline
2.2 Ledger-first enforcement
aucun paiement sans écriture ledger
aucun paiement sans double-entry
2.3 Atomicity rule
Either ALL steps succeed OR NOTHING is committed
3. PAYMENT ARCHITECTURE
3.1 Flow global
Validate request
   ↓
Check balance (Ledger)
   ↓
Risk Engine check
   ↓
FX Engine (if needed)
   ↓
Fee Engine calculation
   ↓
Ledger double-entry write
   ↓
Settlement trigger (if external)
   ↓
Event emission

8.1 Trigger conditions

9. CONCURRENCY CONTROL

9.2 Double-spend protection

10. ERROR HANDLING
Standard format
{
  "error": {
    "code": "PAYMENT_003",
    "message": "Insufficient funds",
    "traceId": "uuid"
  }
}
Error categories
PAYMENT_001 → invalid request
PAYMENT_002 → risk blocked
PAYMENT_003 → insufficient funds
PAYMENT_004 → FX failure
PAYMENT_005 → ledger failure
