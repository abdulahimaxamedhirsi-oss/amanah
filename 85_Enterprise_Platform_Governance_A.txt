(V3 — Enterprise Platform Governance, Architecture & Technology Standards Framework)
1. OBJECTIF

Ce document définit le cadre officiel de gouvernance technique d'Amanah.

Il couvre :

Enterprise Architecture Governance ;
Technology Standards ;
Architecture Decision Records (ADR) ;
API Governance ;
Data Governance ;
AI Governance ;
Cloud Governance ;
Change Management ;
Technical Debt ;
Lifecycle Management.

L'objectif est de garantir la cohérence, la qualité et la pérennité de la plateforme.

2. CORE PRINCIPLES
2.1 Architecture Before Implementation
Every major technical decision must be reviewed before implementation.
2.2 Standards Over Preferences
Technology choices follow enterprise standards rather than individual preferences.
2.3 Long-Term Maintainability
Maintainability is more important than short-term optimization.

Claude Code must:

NEVER introduce unapproved technologies without an ADR.
ALWAYS document architectural decisions.
ALWAYS respect governance workflows.
ALWAYS maintain technical documentation.
ALWAYS assess risks before major changes.
ALWAYS manage technical debt proactively.

27. SUCCESS CRITERIA

La gouvernance est conforme lorsque :

toutes les décisions majeures sont documentées ;
les standards techniques sont appliqués de manière cohérente ;
les exceptions sont contrôlées ;
la dette technique est maîtrisée ;
la plateforme évolue sans perte de cohérence.
