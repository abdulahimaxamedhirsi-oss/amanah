1. OBJECTIF DU SYSTÈME

Le Card System d’Amanah est une infrastructure complète permettant d’émettre, gérer et sécuriser des cartes de paiement liées directement au ledger financier central.

👉 Objectif :

permettre paiements globaux (Visa/Mastercard rails abstraction)
connecter wallet ↔ card via ledger immutable
sécuriser chaque transaction en temps réel
gérer fraude, limites et conformité
supporter scale fintech mondiale
2. CORE PRINCIPLE (ABSOLUTE RULE)
2.1 Card is NOT money storage
A card is only a payment interface linked to wallet balance via ledger.
No balance exists on the card itself.
2.2 Ledger supremacy
every card transaction MUST go through ledger validation
no bypass possible
no offline approval finality
2.3 Authorization model

Card system is:

Wallet → Ledger → Card Authorization Layer → External Network

4. CARD LIFECYCLE (FULL STATE MACHINE)
CREATED
  ↓
TOKENIZATION_PENDING
  ↓
ACTIVE
  ↓
SUSPENDED (temporary block)
  ↓
FROZEN (security lock)
  ↓
TERMINATED (irreversible)
4.1 State rules
ACTIVE → usable
SUSPENDED → retry possible
FROZEN → security investigation
TERMINATED → permanent disable

6. CARD TOKENIZATION SYSTEM
6.1 Security model
PAN never stored in raw form
token stored in secure vault
vault encryption AES-256
6.2 Flow
Card data → Tokenization Service → Secure Vault → Payment Network
6.3 Token properties
single merchant restriction optional
time-based expiration optional
device binding optional
7. CARD AUTHORIZATION ENGINE
7.1 Full flow
1. Merchant sends request
2. Card token validated
3. Risk engine evaluation
4. Wallet balance checked via ledger
5. Limits validated
6. Approval or rejection returned
7.2 Decision matrix
Condition	Result
insufficient balance	decline
high risk score	decline
limit exceeded	decline
all valid	approve
8. LIMIT SYSTEM (VERY IMPORTANT)
8.1 Types of limits
per transaction limit

merchant-specific limit
country-based restriction
8.2 Enforcement rule
Limits are enforced BEFORE ledger authorization.

9.1 Detection signals

9.2 Fraud scoring model
0–30 → safe

70–90 → restricted
90–100 → blocked
9.3 Actions
