2. CORE PRINCIPLES
2.1 Cloud Native First

2.2 Automation Everywhere

Infrastructure, deployment and operations should be automated whenever possible.

Security controls must be embedded into every infrastructure layer.

33. CLAUDE CODE RULES

Claude Code must:

ALWAYS design cloud-native architectures.
ALWAYS use Infrastructure as Code for infrastructure changes.
NEVER deploy production changes without security validation.
ALWAYS implement automated testing pipelines.
ALWAYS protect secrets using approved secret management systems.
ALWAYS maintain environment separation.
ALWAYS include observability in every production service.
ALWAYS design for high availability and disaster recovery.
ALWAYS follow Zero Trust security principles.
ALWAYS follow Amanah Enterprise Cloud & DevOps Standards.
34. SUCCESS CRITERIA

Le framework Cloud & DevOps est conforme lorsque :

l'infrastructure peut évoluer automatiquement ;
les déploiements sont rapides et sécurisés ;
les incidents sont détectés et traités rapidement ;
les données critiques restent protégées ;
la plateforme atteint un niveau bancaire de disponibilité.
