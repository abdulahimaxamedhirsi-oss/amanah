1. OBJECTIF

Ce document définit le framework officiel de cybersécurité d'Amanah.

Il couvre :

Zero Trust Architecture ;
Identity & Access Management (IAM) ;
Privileged Access Management (PAM) ;
Multi-Factor Authentication (MFA) ;
Encryption ;
Key Management ;
Security Operations Center (SOC) ;
SIEM/SOAR ;
Threat Intelligence ;
Incident Response ;
Cyber Resilience.

L'objectif est de protéger la banque contre les cybermenaces tout en garantissant disponibilité, confidentialité et intégrité.

2. CORE PRINCIPLES
2.1 Zero Trust Security

Never trust.
Always verify.

Every user, device, application and request must be continuously authenticated and authorized.

2.2 Defense in Depth

Multiple independent security layers protect critical banking assets.

2.3 Security by Design

Security controls must be embedded from architecture to deployment.

4. ZERO TRUST MODEL

Chaque accès est évalué selon :

identité ;
appareil ;
localisation ;
comportement ;
risque ;
contexte.

Aucune connexion n'est considérée comme automatiquement fiable.

34. SUCCESS CRITERIA

Le framework Cybersecurity est conforme lorsque :

tous les accès sont vérifiés ;
les données sensibles sont protégées ;
les incidents sont détectés rapidement ;
les opérations critiques sont sécurisées ;
la banque peut continuer son activité malgré une cyberattaque.
