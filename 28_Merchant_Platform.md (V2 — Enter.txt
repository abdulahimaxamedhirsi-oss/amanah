28_Merchant_Platform.md (V2 — Enterprise Merchant & Business Payment System)
1. OBJECTIF DU SYSTÈME

Ce module définit la plateforme marchands d’Amanah.

👉 Objectif :

permettre aux entreprises d’accepter des paiements
gérer des flux entrants/sortants
fournir API type Stripe
connecter e-commerce + agents + mobile money
automatiser les settlements
2. CORE PRINCIPLE
2.1 Merchant = financial actor
Merchant is not a user, but a financial entity with ledger exposure
2.2 Ledger always applies
tous les paiements marchands passent par ledger
aucune transaction directe hors système
2.3 Platform as infrastructure

Amanah devient :

wallet system (B2C)
payment infrastructure (B2B)
