68_Cloud_Infrastructure_Standards.md
(V3 — Enterprise Cloud Infrastructure & Platform Standards)
1. OBJECTIF

Ce document définit les standards d'infrastructure cloud obligatoires pour Amanah.

Il couvre :

architecture Cloud Native ;
Kubernetes ;
multi-cloud ;
réseau ;
stockage ;
calcul ;
observabilité ;
résilience ;
sécurité ;
FinOps.

Tous les environnements Amanah doivent respecter ces standards.

2. CORE PRINCIPLES
2.1 Cloud Native First
Applications must be designed for cloud-native environments.

Les services sont conteneurisés, stateless lorsque possible et orchestrés.

2.2 High Availability
No single point of failure is acceptable.
2.3 Infrastructure as Code
All infrastructure must be version controlled and reproducible.

4. CLOUD PROVIDERS

Amanah est compatible avec :

Amazon Web Services (AWS)
Microsoft Azure
Google Cloud Platform (GCP)
OpenShift
VMware Tanzu
Kubernetes On-Premise

Aucun fournisseur ne doit être imposé par l'architecture.

5. MULTI-CLOUD STRATEGY

Objectifs :

éviter le verrouillage fournisseur ;
assurer la continuité d'activité ;
respecter les réglementations locales ;
optimiser les coûts.

Modes supportés :

Single Cloud
Active / Passive Multi-Cloud
Active / Active Multi-Cloud
6. KUBERNETES STANDARDS

10. STORAGE

Types de stockage :

Block Storage

11. COMPUTE

CPU Requests/Limits

12. AUTOSCALING

Déclencheurs :

CPU
Mémoire
Nombre de requêtes
Files de messages
Métriques métier

L'autoscaling ne doit jamais compromettre les performances du Ledger.

13. HIGH AVAILABILITY

Toutes les applications critiques doivent fonctionner sur plusieurs zones de disponibilité.

Objectifs :

aucune interruption lors de la perte d'un nœud ;
redémarrage automatique ;
réplication des données.
14. DISASTER RECOVERY

Chaque environnement définit :

RPO
RTO

15. BACKUP STRATEGY

25. CLAUDE CODE RULES

Claude Code must:

ALWAYS design cloud-native services.
NEVER introduce single points of failure.
ALWAYS deploy workloads through Kubernetes.
ALWAYS encrypt infrastructure communications.
ALWAYS support Infrastructure as Code.
NEVER hardcode infrastructure configuration.
ALWAYS define resource requests and limits.
ALWAYS consider disaster recovery.
ALWAYS optimize for scalability and resilience.
ALWAYS follow Enterprise Cloud Infrastructure Standards.
26. SUCCESS CRITERIA

Le standard Cloud Infrastructure est respecté lorsque :

l'infrastructure est entièrement automatisée ;
les services sont hautement disponibles ;
les déploiements sont reproductibles ;
la plateforme peut évoluer horizontalement ;
les coûts, la sécurité et la conformité sont maîtrisés.
