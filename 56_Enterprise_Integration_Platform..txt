1. OBJECTIF DU SYSTÈME

L'Enterprise Integration Platform (EIP) est la couche d'intégration universelle d'Amanah.

Elle permet de connecter Amanah à :

banques
banques centrales
réseaux de paiement
Mobile Money
Open Banking
ERP
CRM
fournisseurs KYC
fournisseurs AML
fournisseurs FX
systèmes internes
plateformes partenaires

Aucun service métier ne communique directement avec un système externe.

Toutes les intégrations transitent par l'EIP.

2. CORE PRINCIPLES
2.1 Integration by Abstraction
Business services never communicate directly with external providers.
2.2 Provider Independence
External providers can be replaced without changing business logic.
2.3 Event Driven Integration
Integrations are triggered by business events whenever possible.

22. CONNECTOR SDK

Chaque connecteur implémente une interface standard :

interface IntegrationConnector {
  connect(): Promise<void>;
  authenticate(): Promise<void>;
  execute(request): Promise<Response>;
  healthCheck(): Promise<Status>;
  disconnect(): Promise<void>;
}

Cette interface garantit l'interchangeabilité des fournisseurs.

23. CLAUDE CODE RULES

Claude Code must:

NEVER allow business services to call external providers directly.
ALWAYS route external communication through the Enterprise Integration Platform.
ALWAYS support provider abstraction.
NEVER hardcode provider-specific logic in business services.
ALWAYS implement retry and circuit breaker mechanisms.
ALWAYS encrypt credentials and API keys.
ALWAYS log external interactions with traceability.
ALWAYS support connector versioning.
NEVER expose provider-specific errors directly to end users.
ALWAYS design connectors to be replaceable without affecting business workflows.
24. SUCCESS CRITERIA

L'Enterprise Integration Platform est considérée comme complète lorsque :

toutes les communications externes transitent par une couche d'intégration unique ;
les fournisseurs peuvent être remplacés sans modifier les services métier ;
les principaux réseaux bancaires, cartes, Mobile Money, ERP et CRM sont supportés ;
les intégrations sont sécurisées, observables et résilientes ;
les connecteurs sont standardisés, versionnés et facilement extensibles.
