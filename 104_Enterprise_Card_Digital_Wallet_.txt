104_Enterprise_Card_Digital_Wallet_Framework.md
(V3 — Enterprise Card Management, Digital Wallet, Tokenization & Wallet-as-a-Service Framework)
1. OBJECTIF

Ce document définit le framework officiel de gestion des cartes et portefeuilles numériques d'Amanah.

Il couvre :

Card Management System (CMS) ;
Debit Cards ;
Credit Cards ;
Prepaid Cards ;
Virtual Cards ;
Disposable Cards ;
Digital Wallets ;
Tokenization ;
Card Controls ;
Wallet-as-a-Service (WaaS).

L'objectif est de fournir une plateforme de cartes moderne, temps réel, hautement sécurisée et extensible.

2. CORE PRINCIPLES
2.1 Digital First

Cards should be available digitally before physical delivery whenever possible.

2.2 Security by Design

Every card operation must be authenticated, encrypted and auditable.

2.3 Real-Time Control

Customers must control their cards instantly.

Chaque carte peut être représentée par plusieurs tokens.

27. CLAUDE CODE RULES

Claude Code must:

ALWAYS tokenize sensitive card information.
NEVER expose PAN or CVV outside authorized secure components.
ALWAYS support real-time card controls.
ALWAYS reconcile card transactions with the Ledger.
ALWAYS integrate fraud screening before authorization.
ALWAYS maintain immutable card audit trails.
ALWAYS support virtual and disposable cards.
ALWAYS expose card services through secure APIs.
ALWAYS support wallet token lifecycle management.
ALWAYS follow Amanah Enterprise Card Standards.
28. SUCCESS CRITERIA

Le framework Cartes est conforme lorsque :

les cartes peuvent être émises et activées en temps réel ;
les contrôles clients sont instantanés ;
toutes les transactions sont rapprochées avec le Ledger ;
les données sensibles sont tokenisées et protégées ;
les intégrations avec les wallets numériques sont sécurisées et conformes.
