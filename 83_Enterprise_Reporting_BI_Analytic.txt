1. OBJECTIF

Ce document définit l'architecture officielle de reporting et d'analytics d'Amanah.

Il couvre :

Enterprise Reporting ;
Business Intelligence (BI) ;
Executive Dashboards ;
Data Warehouse ;
Data Lake ;
Real-Time Analytics ;
Regulatory Reporting ;
Financial Reporting ;
AI Analytics ;
Self-Service Analytics.

L'objectif est de fournir une plateforme de données unique pour l'ensemble de l'écosystème Amanah.

2. CORE PRINCIPLES
2.1 Single Source of Analytical Truth
Operational systems execute transactions.
Analytical systems produce insights.

Les systèmes analytiques ne remplacent jamais le Ledger.

2.2 Near Real-Time Analytics
Business insights should be available with minimal delay.
2.3 Governed Data
Every analytical dataset must have an owner, lineage and quality controls.

25. CLAUDE CODE RULES

Claude Code must:

NEVER use analytical systems as the operational source of truth.
ALWAYS preserve data lineage.
ALWAYS validate data quality.
ALWAYS version regulatory reports.
ALWAYS isolate tenant analytics.
ALWAYS secure sensitive analytical data.
ALWAYS support near real-time reporting.
ALWAYS maintain KPI consistency.
ALWAYS integrate analytics through governed pipelines.
ALWAYS follow Enterprise Analytics Standards.
26. SUCCESS CRITERIA

Le framework Analytics est conforme lorsque :

les données sont gouvernées et traçables ;
les KPI sont cohérents dans toute la plateforme ;
les tableaux de bord sont disponibles en quasi temps réel ;
les rapports réglementaires sont reproductibles ;
les analyses IA sont explicables et sécurisées.
