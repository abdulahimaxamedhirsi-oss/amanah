(V3 — Enterprise Plugin, SDK, Marketplace & Extensibility Framework)
1. OBJECTIF

Ce document définit le framework officiel d'extensibilité d'Amanah.

Il couvre :

Plugin Architecture ;
SDK ;
Marketplace ;
Extensions ;
Event Hooks ;
UI Extensions ;
API Extensions ;
Sandboxing ;
Certification ;
Lifecycle des plugins.

L'objectif est de permettre aux partenaires d'étendre Amanah sans modifier le Core Banking.

2. CORE PRINCIPLES
2.1 Core Remains Untouched
Plugins must never modify the Core Banking Platform directly.
2.2 Secure by Design
Every extension must execute within a controlled security boundary.
2.3 Backward Compatibility
Platform upgrades must not break certified plugins.

26. CLAUDE CODE RULES

Claude Code must:

NEVER allow plugins to access the Core Banking database directly.
ALWAYS execute plugins inside isolated runtime environments.
ALWAYS validate plugin signatures before execution.
ALWAYS enforce least-privilege permissions.
ALWAYS version plugin APIs.
ALWAYS expose extension points through supported SDKs.
ALWAYS audit plugin activity.
ALWAYS certify plugins before Marketplace publication.
ALWAYS maintain backward compatibility across SDK versions.
ALWAYS follow Enterprise Plugin Standards.
27. SUCCESS CRITERIA

Le framework Plugin est conforme lorsque :

les extensions sont isolées du Core ;
les plugins utilisent uniquement les interfaces officielles ;
les permissions sont contrôlées ;
les plugins sont signés et certifiés ;
les mises à jour sont compatibles avec les versions précédentes.
