2. CORE PRINCIPLES
2.1 Payment Integrity

Every payment must be processed exactly once.

2.2 Ledger Consistency

26. CLAUDE CODE RULES

Claude Code must:

ALWAYS guarantee exactly-once payment processing.
ALWAYS post completed payments to the immutable Ledger.
NEVER bypass AML, fraud or sanction screening.
ALWAYS implement idempotency for payment requests.
ALWAYS support ISO 20022 canonical data models.
ALWAYS reconcile payments with settlement records.
ALWAYS maintain immutable payment audit trails.
ALWAYS publish payment lifecycle events.
ALWAYS isolate payment rail integrations behind adapters.
ALWAYS follow Amanah Enterprise Payments Standards.
27. SUCCESS CRITERIA

Le framework de paiements est conforme lorsque :

chaque paiement est traité une seule fois ;
les paiements sont rapprochés automatiquement avec le Ledger ;
les différents rails de paiement sont interchangeables ;
les contrôles AML/Fraud sont appliqués avant l'exécution ;
les paiements sont observables, auditables et traçables de bout en bout.
