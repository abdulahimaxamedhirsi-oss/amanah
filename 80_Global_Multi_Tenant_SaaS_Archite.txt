1. OBJECTIF

Ce document définit l'architecture SaaS multi-tenant d'Amanah.

Il couvre :

Multi-Tenant ;
White Label ;
Data Isolation ;
Regional Deployment ;
Sovereign Cloud ;

L'objectif est de permettre à Amanah d'opérer comme une plateforme bancaire mondiale pouvant servir plusieurs institutions simultanément.

2. CORE PRINCIPLES
2.1 Tenant Isolation
Every tenant must be logically and securely isolated.

Aucune banque ne peut accéder aux données d'une autre.

2.2 Configuration Over Customization
Business differences should be implemented through configuration rather than custom code.
2.3 One Platform, Many Banks
A single platform can securely serve multiple financial institutions.

25. CLAUDE CODE RULES

Claude Code must:

ALWAYS preserve tenant isolation.
NEVER execute queries without tenant context.
ALWAYS implement configuration before customization.
ALWAYS support white-label branding.
ALWAYS isolate identities between tenants.
ALWAYS enforce feature flags.
ALWAYS support regional deployment.
ALWAYS respect data sovereignty rules.
ALWAYS audit tenant administration actions.
ALWAYS follow Enterprise Multi-Tenant SaaS Standards.
26. SUCCESS CRITERIA

L'architecture SaaS est conforme lorsque :

plusieurs banques peuvent utiliser la même plateforme sans partage de données ;
chaque tenant est entièrement configurable ;
les exigences de résidence des données sont respectées ;
les ressources et coûts sont isolés ;
la plateforme peut évoluer horizontalement vers des centaines d'institutions.
