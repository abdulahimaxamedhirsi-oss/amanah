50_Public_API_Platform.md (V3 — Enterprise Public API & Developer Platform)
1. OBJECTIF DU SYSTÈME

La Public API Platform transforme Amanah en plateforme ouverte permettant à des développeurs externes d’intégrer, construire et monétiser des services financiers.

Elle permet :

accès sécurisé aux fonctionnalités bancaires
intégration tierce (fintech, ERP, apps)
automatisation des paiements
création de produits financiers externes
webhooks en temps réel
SDK multi-langage
sandbox développeur
marketplace d’API
2. CORE PRINCIPLE
2.1 API as a Product
Every API endpoint is a product with usage, limits, pricing and lifecycle.
2.2 Zero Trust API Access
Every request must be authenticated, authorized and rate-limited.

Aucune requête n’est fiable par défaut.

2.3 Tenant + Entity Awareness
Every API call is bound to tenantId and entityId.

18. CLAUDE CODE RULES

Claude Code must:

ALWAYS require authentication for every API call.
NEVER bypass API Gateway.
ALWAYS enforce tenantId and entityId context.
NEVER expose internal Ledger structures via API.
ALWAYS enforce rate limiting before processing.
NEVER allow API access to secrets or private keys.
ALWAYS support versioning and backward compatibility.
ALWAYS log every request with traceId.
NEVER allow sandbox data leakage into production.
ALWAYS apply zero-trust principles.
19. SUCCESS CRITERIA

La Public API Platform est considérée comme complète lorsque :

des développeurs externes peuvent intégrer Amanah en quelques minutes ;
les API sont sécurisées, versionnées et observables ;
un système de sandbox permet le développement sans risque ;
les API peuvent être monétisées ;
les webhooks assurent une communication temps réel fiable ;
la plateforme supporte des millions de requêtes par jour sans impact sur le core banking.
