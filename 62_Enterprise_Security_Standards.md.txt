62_Enterprise_Security_Standards.md (V3 — Enterprise Security Standards)
1. OBJECTIF

Ce document définit les standards de sécurité obligatoires pour l'ensemble de la plateforme Amanah.

Il couvre :

l'authentification ;

l'autorisation ;

le chiffrement ;

la gestion des secrets ;

la sécurité API ;

la sécurité des données ;

la sécurité cloud ;

la sécurité IA ;

la réponse aux incidents.

Tous les composants Amanah doivent respecter ce standard.

2. CORE PRINCIPLES
Security by Design
Mandatory

La sécurité est intégrée dès la conception.

Elle n'est jamais ajoutée après coup.

Zero Trust
Always Verify

Never trust. Always verify.

Least Privilege
Minimum Access

Every user, service and AI agent receives only the minimum required access.

3. IDENTITY & AUTHENTICATION

Standards obligatoires :

OAuth2 + OpenID Connect

MFA obligatoire pour les opérations sensibles

WebAuthn / Passkeys supportés

Sessions courtes et renouvelables

Détection d'anomalies de connexion

Exigence critique
Obligatoire

Aucun accès administrateur sans MFA.

4. AUTHORIZATION

Le système combine :

RBAC

Role-Based Access Control

Rôles, groupes, héritage et permissions standardisées.

Exemple : Admin → Manager → Operator → Viewer

ABAC

Attribute-Based Access Control

Décisions basées sur l'utilisateur, la ressource, le pays, l'heure, le risque et le tenant.

Exemple : accès autorisé uniquement depuis le pays de résidence

6. API SECURITY

Toutes les API doivent :

passer par l'API Gateway ;

être authentifiées ;

être autorisées ;

être rate-limit ;

être journalisées ;

supporter la rotation des clés ;

valider toutes les entrées.

7. SECRET MANAGEMENT

Les secrets sont :

stockés dans un gestionnaire dédié ;

chiffrés ;

rotés automatiquement ;

auditables ;

isolés par tenant et environnement.

8. NETWORK SECURITY

segmentation par environnement.

16. CLAUDE CODE RULES

Claude Code must:

ALWAYS enforce Zero Trust authentication.

NEVER store secrets in source code.

ALWAYS encrypt sensitive data.

NEVER bypass RBAC or ABAC checks.

ALWAYS validate all external input.

ALWAYS log security-relevant events.

NEVER expose internal errors to users.

ALWAYS rotate credentials and tokens.

ALWAYS require MFA for privileged access.

ALWAYS follow these Enterprise Security Standards.

17. SUCCESS CRITERIA

Le standard est respecté lorsque :

toutes les communications sont chiffrées ;

aucun secret n'est présent dans le code ;

RBAC et ABAC sont appliqués partout ;

les audits sont complets et immuables ;

les contrôles OWASP, PCI DSS et ISO 27001 sont satisfaits ;

les incidents de sécurité peuvent être détectés, investigués et traités rapidement.
