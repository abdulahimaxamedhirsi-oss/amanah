1. OBJECTIF DU SYSTÈME ICAN

Le système ICAN (International Customer Account Number) est :

Une identité financière universelle unique pour chaque utilisateur Amanah.

Rôle principal

ICAN permet de :

identifier un utilisateur globalement
connecter tous les rails financiers (mobile money, banque, card, crypto future)
mapper un utilisateur à son Ledger Account
simplifier les paiements internationaux
2. PRINCIPLE OF ICAN DESIGN
2.1 Universal identity layer
ICAN = unique financial identity across all systems
2.2 One user = one ICAN
strict 1:1 mapping
immutable once created
never reused
2.3 Country abstraction

ICAN is independent of:

country
currency
payment rail
3. ICAN STRUCTURE
3.1 Format
AMH-XXXX-XXXX-XXXX

Where:

AMH = Amanah prefix
XXXX = cryptographic segments
3.2 Example
AMH-7F3K-92LD-4PXA
3.3 Properties
globally unique
non-guessable
deterministic generation
4. ICAN GENERATION SYSTEM
4.1 Flow
User created
   ↓
User verified (KYC required)
   ↓
Generate ICAN seed
   ↓
Hash + entropy generation
   ↓
Check uniqueness in DB
   ↓
Persist ICAN
   ↓
Link to Ledger Account
4.2 Rules
ICAN cannot be generated before KYC
ICAN cannot be modified
ICAN must be collision-proof
5. ICAN ↔ LEDGER MAPPING
5.1 Core mapping
ICAN → Ledger Account (1:1)
5.2 Ledger dependency

Every ICAN is tied to:

account_id (ledger)
currency wallets
transaction history
6. ICAN USE CASES
6.1 Send money

bank account

6.2 Receive money

payment address

6.3 Cross-border payments

ICAN abstracts:

FX complexity

cryptographically random
