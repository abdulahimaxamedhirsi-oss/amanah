(V3 — Enterprise Web Portal, Operations Console, Administration & Backoffice Framework)
1. OBJECTIF

Ce document définit le framework officiel du portail Web et du Backoffice d'Amanah.

Il couvre :

Customer Web Portal ;
Employee Portal ;
Operations Console ;
Compliance Console ;
Risk Dashboard ;
Administration Portal ;
Workflow Management ;
Audit Console ;
Reporting ;
Identity & Access Management.

L'objectif est de fournir une plateforme web unifiée, sécurisée et orientée productivité.

2. CORE PRINCIPLES
2.1 Single Workspace

Employees should access all operational capabilities from a unified platform.

2.2 Least Privilege

Users shall only access information required for their role.

2.3 Operational Transparency

All administrative actions must be visible, traceable and auditable.

27. CLAUDE CODE RULES

Claude Code must:

ALWAYS enforce RBAC and ABAC for every operation.
NEVER expose customer data beyond authorized permissions.
ALWAYS record administrative actions in immutable audit logs.
ALWAYS support configurable approval workflows.
ALWAYS provide a unified user experience across operational modules.
ALWAYS secure administrator sessions with MFA and session controls.
ALWAYS expose operational capabilities through modular APIs.
ALWAYS optimize dashboards and searches for enterprise-scale performance.
ALWAYS integrate observability into all backoffice modules.
ALWAYS follow Amanah Enterprise Backoffice Standards.
28. SUCCESS CRITERIA

Le framework Web Portal & Backoffice est conforme lorsque :

les collaborateurs disposent d'un espace de travail unifié ;
les contrôles d'accès sont appliqués de manière cohérente ;
toutes les actions sont traçables et auditables ;
les workflows d'approbation sont configurables ;
les performances restent élevées même à grande échelle.
