08_Backend_Architecture.md (V2 — Enterprise Fintech Core System)
1. OBJECTIF DU DOCUMENT

Ce document définit l’architecture backend complète d’Amanah.

👉 Il doit permettre de construire un système :

fintech-grade
ledger-safe
scalable
event-driven
sans ambiguïté d’exécution
2. ARCHITECTURE OVERVIEW

Amanah backend est un :

Modular Monolith Event-Driven System (phase 1)
évolutif vers microservices (phase 2)

2.1 Architecture globale
API Layer (NestJS Controllers)
        ↓
Service Layer (Business Logic)
        ↓
Domain Layer (Ledger / FX / Payments)
        ↓
Event Bus (Redis / Kafka)
        ↓
Infrastructure Layer (DB / External APIs)
3. CORE DESIGN PRINCIPLE
3.1 Ledger-Centric Architecture
ALL financial operations → MUST pass through Ledger Engine
3.2 No direct wallet updates

Wallet = projection only

3.3 Event-driven system

Every important action emits events:

payment.created
ledger.entry.created
fx.rate.locked

6. LEDGER ENGINE (CORE SYSTEM)
6.1 Principle
Ledger = SINGLE SOURCE OF FINANCIAL TRUTH
6.2 Structure

Each transaction produces:

debit entries
credit entries
6.3 Entry format
{
  "id": "uuid",
  "accountId": "string",
  "type": "DEBIT | CREDIT",
  "amount": 100,
  "currency": "USD",
  "transactionId": "uuid",
  "timestamp": "ISO",
  "traceId": "uuid"
}
6.4 Rule
SUM(DEBIT) = SUM(CREDIT)
6.5 Immutability
no update
no delete
only append
7. PAYMENT ENGINE
7.1 Responsibility
orchestrate payments

write to ledger
7.2 Flow

   ↓

   ↓

   ↓

   ↓

   ↓
