(V3 — Enterprise API Economy, Open Banking, Open Finance & Banking-as-a-Service Framework)
1. OBJECTIF

Ce document définit le framework officiel API Economy d'Amanah.

Il couvre :

API-First Architecture ;
Open Banking ;
Open Finance ;
Banking-as-a-Service (BaaS) ;
Embedded Finance ;
API Marketplace ;
Developer Portal ;
Consent Management ;
Partner Ecosystem ;
API Monetization.

L'objectif est de transformer Amanah en une plateforme bancaire ouverte, extensible et sécurisée.

2. CORE PRINCIPLES
2.1 API First

Every banking capability must be exposed through secure and documented APIs.

2.2 Open by Design

External integrations must be designed as first-class citizens.

2.3 Secure by Default

Every API request must be authenticated, authorized and auditable.

27. CLAUDE CODE RULES

Claude Code must:

ALWAYS expose business capabilities through secure APIs.
NEVER bypass API Gateway security policies.
ALWAYS version public APIs.
ALWAYS require explicit customer consent where applicable.
ALWAYS support OAuth 2.1 and OpenID Connect.
ALWAYS implement idempotency for financial operations.
ALWAYS document APIs using OpenAPI specifications.
ALWAYS maintain immutable API audit logs.
ALWAYS publish versioned event schemas.
ALWAYS follow Amanah API Governance Standards.
28. SUCCESS CRITERIA

Le framework API Economy est conforme lorsque :

toutes les capacités bancaires sont accessibles via API ;
les partenaires disposent d'un portail développeur complet ;
les consentements clients sont respectés ;
les APIs sont sécurisées, versionnées et observables ;
les intégrations sont découplées du Core Banking.
