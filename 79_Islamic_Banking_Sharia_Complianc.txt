1. OBJECTIF

Ce document définit le framework officiel de finance islamique d'Amanah.

Il couvre :

gouvernance Sharia ;
contrats islamiques ;
moteur de conformité ;
workflows d'approbation ;
Zakat ;
purification des revenus ;
Sukuk ;
investissements ;
audit Sharia ;
conformité AAOIFI et IFSB.

L'objectif est de permettre à Amanah d'opérer comme une plateforme bancaire conforme aux principes de la finance islamique.

2. CORE PRINCIPLES
2.1 Sharia by Design
Islamic compliance must be enforced by the platform architecture.

La conformité est intégrée dans les règles métier.

2.2 No Riba
Interest-based financial operations are prohibited.

Toute opération impliquant des intérêts est bloquée ou signalée selon la politique de l'institution.

2.3 Ethical Finance
Financial products must promote fairness, transparency and shared risk.

4. SHARIA GOVERNANCE

Chaque institution peut définir :

son Sharia Board ;
ses politiques internes ;
ses interprétations juridiques ;
ses exceptions documentées.

Le système ne remplace jamais les décisions du Sharia Board.

5. SHARIA BOARD

Composition :

Chairman
Sharia Scholars
Compliance Officer
Internal Auditor
Product Representatives

Responsabilités :

validation des produits ;
validation des contrats ;
revue des exceptions ;
audits périodiques.
6. SHARIA RULE ENGINE

Le moteur applique automatiquement les règles.

Transaction

↓

25. INTEGRATION WITH CORE BANKING

Toutes les opérations islamiques utilisent :

le Ledger ;

le moteur documentaire ;
le moteur de conformité ;
le moteur d'audit.

Il n'existe qu'un seul Core Banking.

26. CLAUDE CODE RULES

Claude Code must:

ALWAYS enforce configurable Sharia compliance rules.
NEVER calculate interest for Islamic financial products.
ALWAYS preserve complete audit trails.
ALWAYS separate Sharia policies from business implementation.
ALWAYS support institution-specific configuration.
ALWAYS integrate Islamic products with the Ledger.
ALWAYS maintain immutable contract history.
ALWAYS support configurable Zakat calculations.
ALWAYS require governance workflows where configured.
ALWAYS follow Enterprise Islamic Banking Standards.
27. SUCCESS CRITERIA

Le framework de finance islamique est conforme lorsque :

les produits sont gérés par des workflows dédiés ;
les règles de conformité sont configurables et appliquées automatiquement ;
toutes les décisions sont auditables ;
les calculs et rapports requis sont disponibles ;
la gouvernance du Sharia Board est intégrée à la plateforme.
