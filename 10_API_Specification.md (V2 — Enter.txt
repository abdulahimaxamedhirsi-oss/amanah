10_API_Specification.md (V2 — Enterprise Fintech API Contract)
1. OBJECTIF DU DOCUMENT

Ce document définit toutes les APIs backend d’Amanah :

endpoints REST
payloads exacts
règles d’authentification
règles d’idempotency
erreurs standardisées
lien direct avec Ledger

👉 Ce document = contrat officiel entre Frontend ↔ Backend ↔ Ledger

2. GLOBAL API PRINCIPLES
2.1 Ledger-first rule
Aucune API ne peut exister sans impact ledger traçable (direct ou indirect)
2.2 No silent success / no silent failure

Chaque réponse doit être explicite :

success
failed
pending
2.3 Idempotency mandatory

Toutes les APIs financières doivent accepter :

Idempotency-Key: UUID
2.4 Traceability

Chaque request doit contenir :

traceId
