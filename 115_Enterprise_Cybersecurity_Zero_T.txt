L'objectif est de protéger Amanah contre les cybermenaces modernes tout en assurant disponibilité, confidentialité et intégrité.

2. CORE PRINCIPLES
2.1 Zero Trust

Never trust.
Always verify.

Every user, device, application and request must be continuously validated.

2.2 Defense in Depth

Multiple security layers must protect every critical asset.

2.3 Security by Design

Security controls must exist from architecture design to production operation.

30. CLAUDE CODE RULES

Claude Code must:

ALWAYS apply security-by-design principles.
NEVER bypass authentication or authorization controls.
ALWAYS validate identity and permissions before operations.
ALWAYS protect secrets and cryptographic keys.
ALWAYS generate secure code following banking security standards.
ALWAYS include security testing in development workflows.
ALWAYS maintain security audit trails.
ALWAYS avoid exposing sensitive financial information.
ALWAYS follow Zero Trust principles.
ALWAYS follow Amanah Enterprise Cybersecurity Standards.
31. SUCCESS CRITERIA

Le framework Cybersecurity est conforme lorsque :

tous les accès sont vérifiés ;
les données sensibles sont protégées ;
les menaces sont détectées rapidement ;
les incidents peuvent être contenus efficacement ;
les opérations bancaires restent disponibles même sous attaque.
