(V3 — Enterprise Globalization, Localization, Multi-Country & Internationalization Framework)
1. OBJECTIF

Ce document définit le framework officiel de mondialisation d'Amanah.

Il couvre :

Multi-Country ;
Multi-Tenant ;
Multi-Language ;
Multi-Currency ;
Multi-Timezone ;
Internationalization (i18n) ;
Localization (l10n) ;
Réglementation locale ;
Fiscalité ;
Calendriers bancaires.

L'objectif est de permettre à une seule plateforme de fonctionner dans plusieurs pays tout en respectant les spécificités locales.

2. CORE PRINCIPLES
2.1 Country Configuration Over Custom Code
Country-specific behavior must be driven by configuration rather than code changes.
2.2 Global Core, Local Compliance
Maintain one global platform while supporting local legal and regulatory requirements.
2.3 Localization by Design
Every customer-facing interface must support localization.

25. CLAUDE CODE RULES

Claude Code must:

NEVER hardcode country-specific logic.
ALWAYS use localization resources.
ALWAYS support Unicode.
ALWAYS store timestamps in UTC.
ALWAYS version country configurations.
ALWAYS separate global business logic from local regulations.
ALWAYS support multiple currencies.
ALWAYS respect RTL layouts where applicable.
ALWAYS validate country-specific regulatory rules.
ALWAYS follow Enterprise Globalization Standards.
26. SUCCESS CRITERIA

Le framework de mondialisation est conforme lorsque :

une seule plateforme sert plusieurs pays ;
les réglementations locales sont configurables ;
les langues et devises sont gérées dynamiquement ;
les calendriers bancaires sont respectés ;
les interfaces s'adaptent automatiquement aux paramètres régionaux.
