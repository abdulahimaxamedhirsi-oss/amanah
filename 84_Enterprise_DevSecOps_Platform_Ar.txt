1. OBJECTIF

Ce document définit l'architecture officielle DevSecOps d'Amanah.

Il couvre :

L'objectif est de garantir des livraisons rapides, sécurisées, reproductibles et conformes aux exigences bancaires.

2. CORE PRINCIPLES
2.1 Everything as Code
Infrastructure, applications, security policies and deployments must be managed as code.
2.2 Security by Default
Security controls must be embedded into every stage of delivery.
2.3 Git is the Source of Truth
All production changes originate from version-controlled repositories.

25. CLAUDE CODE RULES

Claude Code must:

ALWAYS manage infrastructure as code.
NEVER deploy directly to production outside approved workflows.
ALWAYS require automated testing before deployment.
ALWAYS perform security scanning in CI/CD.
ALWAYS use GitOps for production deployments.
ALWAYS sign release artifacts.
ALWAYS protect secrets using a dedicated secret manager.
ALWAYS support automated rollback.
ALWAYS maintain complete deployment audit trails.
ALWAYS follow Enterprise DevSecOps Standards.
26. SUCCESS CRITERIA

La plateforme DevSecOps est conforme lorsque :

tous les changements passent par Git ;
chaque déploiement est reproductible ;
les contrôles de sécurité sont automatisés ;
les artefacts sont signés et traçables ;
les déploiements peuvent être annulés rapidement et en sécurité.
